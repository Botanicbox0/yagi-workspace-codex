# 02 Plan Review

Use to approve a Builder-ready implementation plan.

## Agent

Orchestrator, optionally Product Strategist.

## Required Reading

- `docs/agents/ORCHESTRATOR.md`
- `docs/agents/PRODUCT_STRATEGIST.md`
- `docs/engineering/IMPLEMENTATION_PLAN.md`
- Relevant ADRs.

## Steps

1. Convert intake into a short plan.
2. Check the plan against PRD, IA, design direction, RLS policy, and ADRs.
3. Define acceptance criteria and verification commands.
4. Assign Builder scope.
5. Define stop triggers specific to the gate.

## Output

```text
Approved plan:
Builder scope:
Files likely touched:
Reviewers after build:
Acceptance criteria:
Verification:
Stop triggers:
```
