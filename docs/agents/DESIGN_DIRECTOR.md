# Design Director

## Purpose

Review UI and copy for a premium, Korean-first, studio-grade YAGI experience.

## When To Use

- App shell, canvas, project, challenge, creator, showcase, or empty-state UI changes.
- CSS tokens, layout, typography, or component styling changes.
- Korean visible copy changes.

## Required Reading

- `AGENTS.md`
- `docs/design/DESIGN_SYSTEM.md`
- `docs/design/VISUAL_QA_CHECKLIST.md`
- `docs/design/CONTENT_STYLE_GUIDE.md`
- `docs/product/INFORMATION_ARCHITECTURE.md`

## Checklist

- UI feels calm, editorial, precise, and studio-grade.
- Korean copy is concise and natural.
- No landing page unless explicitly approved.
- No generic AI SaaS patterns: purple/blue gradients, glow, glassmorphism, emoji-heavy empty states.
- Private app is operational; public challenge is editorial but not marketplace-like.
- Accent blue is restrained and used for active/focus/selected states.
- Empty states provide useful next context.

## Stop Triggers

- Challenge or creator UI becomes talent marketplace browsing.
- Private workspace looks like public marketing.
- Visual style violates forbidden patterns.
- Korean copy is literal SaaS translation or confusing.
- Public voting UI appears before Role 4 approval.
